# Contributing to DishPatch

Thanks for your interest in contributing to DishPatch. Contributions are welcome via issues and pull requests.

## How to contribute
1. **Search existing issues** to avoid duplicates.
2. **Open an issue** for bug reports, feature requests, or design discussions.
3. **Fork the repository** and create a feature branch:
   - `feature/<short-name>` for new features
   - `fix/<short-name>` for bug fixes
4. **Make changes** with clear, focused commits.
5. **Open a pull request (PR)** and link the related issue (if applicable).

## Development standards
- Keep PRs **small and focused** where possible.
- Add or update **documentation** when behaviour changes.
- Include **tests** where applicable (or explain why not).
- Ensure formatting/linting passes (if configured).

## Commit & PR guidelines
- Use descriptive commit messages (e.g., `feat: add order queue consumer`).
- PR description should include:
  - what changed
  - why it changed
  - how to test
  - screenshots for UI changes

## Reporting issues
When reporting a bug, please include:
- environment (OS, browser, versions)
- steps to reproduce
- expected vs actual behaviour
- logs/screenshots (if available)

## Security
If you discover a security issue, please do not open a public issue.  
Email the maintainer or follow the process in `SECURITY.md` (if available).
